Word Search

Getting an API call onto my website!

As the reader would be able to tell from my recent blog posts, I have delved into frontend development recently. It has been great learning Javascript, HTML and CSS. What has been particularly exciting (and challenging) is figuring out how these three languages interact with one another.
I have had to come to the realisation that they are not three seperate languages - they are inextricably linked and thinking of them as such is helpful in the development process.
The Project
I decided to create a search box on my website which allowed the user to input a word. This word then uses the 'Words API' API to find the definition of the word and display it on the website.
This functionality is fairly simple but it does a few things which I think are really cool:
> It makes an API call. I was eager to test my ability to code this up in Javascript
> It requires user input. This allows me to code something dynamic.
> It appears on my website. This is an interesting, practical way of demonstrating the skills I acquire while also making the blog a little more fun!
The API call
I have coded up projects with an API before and I find them a little bit daunting. The logic of the 'fetch' function left me a bit stumped. The only way to overcome this fear is to tackle it head-on. 
      
        function getWord(word) {
          let api = `https://wordsapiv1.p.rapidapi.com/words/${word}/definitions`;
        
          fetch(api, options)
            .then(function (response) {
              let data = response.json();
              return data;
            })
            .then(function (data) {
              //console.log(data.definitions[0].definition);
              /*for (item in data) {
              console.log(item);
            }*/
        
              let test1 = data.definitions[0].definition;
              return test1;
            })
            .then(function (test1) {
              wordObj.def = test1;
            })
            .then(function () {
              changeHTML();
            })
            .catch(function (err) {
              console.error(err);
            });
        }
The 'fetch' function is what allows us to access the 'Words' API. The '.then' function basically seems to nest functions, allowing us to retain data. In this case, we retrieve the information from the API and store it as a JSON file. This data is then passed to another function which stores what we need from the API - namely, the word's definition. Once this is done, we run a function to update the HTML in order for the webpage to refresh.
Altering HTML
I'm not sure what makes me more excited - using the API or using JavaScript to update the HTML. It might seem simple but as someone with little frontend experience I find the interaction with all the languages fascinating. 
      
        function changeHTML() {
          wordDef.innerHTML = `${wordObj.input}: ${wordObj.def}`;
          document.getElementById("word").style.color = "#fff";
          document.getElementById("word").style.backgroundColor = "#498f61";
        }
changeHTML() does what it says on the tin. The first line of the function updates the actual text using the innerHTML function. Notice that this data is the data we pulled from the 'Words' API.
I then update the color of the text. This is simply to let the user know that their page has updated.
Allowing user input
The final element required would be to allow the user to input a word. This ended up being simpler than I thought it would be. 
      
        function checkEnter(event) {
          if (event.keyCode == 13) {
            let word = document.getElementById("name-input").value;
            wordObj.input = word;
            getWord(wordObj.input);
          }
        }
We simply have a function which checks to see whether the 'enter' key has been pressed. If it has, the function takes in the inputted text and runs our getWord() function from earlier in order to update the page with the word's definition
Conclusion
Ultimately, this project was about demonstrating my ability to create something using Javascript and HTML. This was a learning experience for me and a chance to cut my teeth with the languages, particularly in making an API call. I also learned about user input and updating an HTML document from a Javascript script.
